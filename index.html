Why do some websites show "Refused to connect"?

Security Protection (X-Frame-Options)

Websites like Google, GitHub, YouTube, and Amazon prevent themselves from being loaded inside of an <iframe> on other domains. This protects users from clickjacking attacks, but blocks layout builders like this one.

1

Install a Chrome Header Modifier (Recommended)

You can bypass standard iframe restrictions on local dashboards by installing a browser extension that ignores the X-Frame-Options and Content-Security-Policy headers.

Popular Free Options:
  • Ignore X-Frame-Options (Bypasses security headers for standard local tasks)
  • Allow CORS: Access-Control-Allow-Origin (Bypasses CORS restrictions)
2

Use the Chrome Multi-Window Engine

Instead of displaying the pages inside this grid, use our Chrome Multi-Window Engine (located in the sidebar). It will launch genuine, isolated Chrome windows arranged neatly across your monitors, completely bypassing all embedding issues.

3

Use Embeddable Variants

Many sites offer custom embed pages. For example, instead of loading a YouTube video page, use the YouTube embed link: https://www.youtube.com/embed/VIDEO_ID.
